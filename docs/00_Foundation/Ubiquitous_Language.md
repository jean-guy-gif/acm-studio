# ACM Studio

# Ubiquitous Language

Version : 1.0

Statut : Fondation

Priorité : Critique

---

# Pourquoi ce document existe

ACM Studio repose sur un langage métier unique.

Chaque mot possède une définition officielle.

Tous les développeurs, designers, IA, documentations et interfaces doivent utiliser ces définitions.

Aucun synonyme ne doit être utilisé dans le code ou la documentation.

Ce document est la référence absolue.

---

# Project

Définition

Un Project représente une mission de vente.

Il ne représente jamais un bien immobilier.

Il regroupe toutes les informations nécessaires à la préparation et à la conduite du rendez-vous vendeur.

Contient :

• Subject Property

• Competitors

• Analysis

• Presentation

• Meeting

• Report

---

# Subject Property

Définition

Le bien du vendeur.

Il constitue le point de départ de l'analyse.

Il ne possède jamais :

• d'estimation

• de prix conseillé

• de valeur calculée

Le Subject Property est uniquement descriptif.

---

# Competitor

Définition

Annonce concurrente réellement présente sur le marché.

Un Competitor n'est jamais :

• un bien vendu

• une estimation

• une moyenne

Il provient toujours :

• d'un portail immobilier

• d'une capture

• d'un PDF

• d'une saisie manuelle

---

# Analysis

Définition

Analyse automatique du marché réalisée par ACM Studio.

Elle contient :

• statistiques

• lecture du marché

• scoring

• synthèse

L'Analysis ne produit jamais une estimation.

---

# Presentation

Définition

Présentation utilisée pendant le rendez-vous vendeur.

Elle est composée d'une succession de Slides.

Une Slide = une idée.

---

# Slide

Définition

Plus petite unité de présentation.

Une Slide poursuit toujours un objectif psychologique précis.

Une Slide ne contient jamais plusieurs messages.

---

# Storyboard

Définition

Ordre officiel des Slides.

Le Storyboard décrit :

• l'ordre

• les animations

• les interactions

• les transitions

Le Storyboard est indépendant du design graphique.

---

# Builder

Définition

Mode de préparation.

Le conseiller construit son dossier.

Le Builder est utilisé entre le premier et le second rendez-vous.

Le vendeur n'y a jamais accès.

---

# Live

Définition

Mode Rendez-vous.

Le conseiller anime son rendez-vous.

Le logiciel affiche les Slides.

Le vendeur participe.

Aucune fonction de préparation n'est visible.

---

# Meeting

Définition

Séance interactive entre le conseiller et le vendeur.

Le Meeting commence à la première Slide.

Il se termine à la signature ou à la fin de l'entretien.

---

# Response

Définition

Réponse donnée par le vendeur.

Une Response est toujours enregistrée.

Elle peut être :

• un prix

• un choix

• une sélection

• une note

• un commentaire

---

# Market Reading

Nom français

Lecture du marché

Définition

Interprétation des données du marché.

Elle est proposée par l'IA.

Elle est toujours validée par le conseiller.

---

# Conclusion

Définition

Synthèse finale.

Toujours modifiable.

Toujours validée par le conseiller.

Jamais générée automatiquement sans validation.

---

# Advisor Note

Nom français

Note Conseiller

Définition

Document confidentiel.

Jamais présenté au vendeur.

Contient :

• arguments

• objections

• vigilance

• posture

• stratégie

---

# IPC Engine

Nom affiché

Baromètre de perception

Définition

Moteur analysant les réponses du vendeur pendant le rendez-vous.

Il calcule :

• perception du marché

• perception des prix

• concurrent psychologique principal

• sensibilité au prix

• sensibilité aux prestations

• évolution de la perception

Le vendeur ne voit jamais ces calculs.

---

# Psychological Competitor

Nom français

Concurrent psychologique

Définition

Concurrent que le vendeur considère spontanément comme le plus dangereux.

Il ne correspond pas forcément au concurrent objectivement le plus compétitif.

Il est déterminé par les réponses du vendeur.

---

# Price Gap

Nom français

Écart de perception

Définition

Différence entre :

le prix estimé par le vendeur

et

le prix réel affiché.

Cet écart alimente le Baromètre de perception.

---

# Validation

Définition

Action du conseiller consistant à accepter ou modifier une proposition de l'IA.

Aucune présentation n'est générée sans validation.

---

# Export

Définition

Document généré par ACM Studio.

Formats possibles :

• PowerPoint

• PDF

• Compte rendu

• Rapport

# Glossaire utilisateur

Dans l'interface, les termes sont traduits :

Project → Dossier vendeur

Subject Property → Votre bien

Competitor → Concurrent

Meeting → Rendez-vous vendeur

Presentation → Présentation

Advisor Note → Note conseiller

Market Reading → Lecture du marché

IPC Engine → Baromètre de perception

Le conseiller n'a jamais besoin de voir les noms techniques.
